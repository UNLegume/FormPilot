## 設定例

### 最小構成（必須項目のみ）

spreadsheet と commonData の最低限の項目だけを定義した例。options はすべてデフォルト値が適用される。

```json
{
  "spreadsheet": {
    "url": "https://docs.google.com/spreadsheets/d/XXXXXX/edit",
    "columns": {
      "companyName": "A",
      "submitted": "B",
      "formUrl": "C"
    }
  },
  "commonData": {
    "会社名": "株式会社サンプル",
    "氏名": "山田 太郎",
    "メールアドレス": "info@example.com",
    "電話番号": "03-1234-5678",
    "お問い合わせ内容": "お世話になります。株式会社サンプルの山田と申します。\n貴社のサービスに関しまして、ご相談をさせていただきたくご連絡いたしました。\n詳細についてお打ち合わせの機会をいただけますと幸いです。\nご検討のほど、よろしくお願いいたします。"
  }
}
```

この構成では:
- `sheet` が省略されているため `"シート1"` が使用される
- `options` が省略されているため全てデフォルト値（`confirmBeforeSubmit: true`、`screenshotAfterSubmit: true`、`skipOnError: true`、`maxCompanies: 150`）が適用される

---

### フル構成（全オプション指定）

全てのオプションを明示的に指定した例。commonData に追加情報を含め、セレクトボックス向けの値も設定している。

```json
{
  "spreadsheet": {
    "url": "https://docs.google.com/spreadsheets/d/XXXXXX/edit",
    "columns": {
      "companyName": "A",
      "submitted": "B",
      "formUrl": "C"
    },
    "sheet": "営業リスト2026"
  },
  "commonData": {
    "会社名": "株式会社サンプル",
    "会社フリガナ": "カブシキガイシャサンプル",
    "氏名": "山田 太郎",
    "姓": "山田",
    "名": "太郎",
    "フリガナ（ひらがな）": "やまだ たろう",
    "フリガナ（カタカナ）": "ヤマダ タロウ",
    "メールアドレス": "info@example.com",
    "担当者メールアドレス": "yamada@example.com",
    "情報受信用メールアドレス": "contact@example.com",
    "電話番号": "03-1234-5678",
    "部署名": "営業部",
    "役職": "営業部長",
    "生年月日": "01/01/1990",
    "会社URL": "https://www.example.co.jp/",
    "住所": "東京都千代田区丸の内1-1-1",
    "郵便番号": "100-0005",
    "お問い合わせ種別": "サービスについてのお問い合わせ",
    "お問い合わせ内容": "お世話になります。株式会社サンプルの山田と申します。\n貴社のサービスに関しまして、ご相談をさせていただきたくご連絡いたしました。\n詳細についてお打ち合わせの機会をいただけますと幸いです。\nご検討のほど、よろしくお願いいたします。"
  },
  "options": {
    "confirmBeforeSubmit": true,
    "screenshotAfterSubmit": true,
    "skipOnError": false,
    "maxCompanies": 50,
    "formSearchKeywords": ["お問い合わせ", "Contact"]
  }
}
```

この構成では:
- `sheet` を明示指定しているため「営業リスト2026」シートを参照する
- `お問い合わせ内容` に `\n` を含めており、テキストエリアに改行付きで入力される
- `お問い合わせ種別` はセレクトボックス向けの値で、LLM がフォームの選択肢の中から最も近いものを選ぶ
- `confirmBeforeSubmit: true` により毎回送信前に確認プロンプトが表示される
- `skipOnError: false` によりエラー発生時に処理を中断する（デバッグ時に有効）
- `maxCompanies: 50` により1回の実行で最大50社を処理する

## 業種別カスタマイズ例

config.json の `commonData` と `options.formSearchKeywords` を業種に合わせて変更するだけで、どの業種でも利用できます。

### SES（システムエンジニアリングサービス）

```json
{
  "commonData": {
    "お問い合わせ種別": "協業・パートナー提携",
    "お問い合わせ内容": "お世話になります。株式会社サンプルの山田と申します。\n貴社のSES事業に関しまして、パートナー提携のご相談をさせていただきたくご連絡いたしました。\n詳細についてお打ち合わせの機会をいただけますと幸いです。\nご検討のほど、よろしくお願いいたします。"
  },
  "options": {
    "formSearchKeywords": ["お問い合わせ", "Contact", "パートナー", "協業", "提携", "SES"]
  }
}
```

### 人材紹介

```json
{
  "commonData": {
    "お問い合わせ種別": "人材紹介のご相談",
    "お問い合わせ内容": "お世話になります。株式会社サンプルの山田と申します。\n貴社の採用ニーズに関しまして、人材紹介サービスのご案内をさせていただきたくご連絡いたしました。\n詳細についてお打ち合わせの機会をいただけますと幸いです。\nご検討のほど、よろしくお願いいたします。"
  },
  "options": {
    "formSearchKeywords": ["お問い合わせ", "Contact", "採用", "求人", "人材"]
  }
}
```

### 不動産

```json
{
  "commonData": {
    "お問い合わせ種別": "物件のお問い合わせ",
    "お問い合わせ内容": "お世話になります。株式会社サンプルの山田と申します。\n貴社の取扱物件に関しまして、詳細をお伺いしたくご連絡いたしました。\nご都合のよいときにお打ち合わせの機会をいただけますと幸いです。\nご検討のほど、よろしくお願いいたします。"
  },
  "options": {
    "formSearchKeywords": ["お問い合わせ", "Contact", "物件", "内覧", "資料請求"]
  }
}
```

> **Note:** 上記は `commonData` と `options` の差分のみ記載しています。実際の config.json では `spreadsheet` セクションや、会社名・氏名等の共通項目も必要です。
